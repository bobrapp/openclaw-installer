## Token Policies

### 1. Use Fine-Grained PATs Only
- Classic PATs are prohibited. Use fine-grained PATs with minimum required scopes.
- Each token must be scoped to the specific repository (not organization-wide).

### 2. Maximum Lifetime: 90 Days
- All PATs must have an expiration date set, maximum 90 days.
- Set calendar reminders 7 days before expiry.
- The `GITHUB_TOKEN` auto-generated by Actions is exempt (scoped per workflow run).

### 3. Minimum Privilege
- Every token should have the absolute minimum permissions needed:
  - CI build/test: `contents:read` only
  - Deploy: `contents:read` + `pages:write`
  - Release: `contents:write` (only for the release workflow)
- Never grant `admin` scope to automated tokens.

### 4. No Tokens in Code
- Tokens must never appear in source code, commits, logs, or error messages.
- Use GitHub Actions secrets for all automated workflows.
- Secret scanning with push protection will block accidental commits.

### 5. Rotation Procedure

```bash
# 1. Create new fine-grained PAT at github.com/settings/personal-access-tokens/new
#    - Set repository: bobrapp/openclaw-installer
#    - Set minimum permissions
#    - Set expiry: 90 days from today

# 2. Update GitHub Actions secrets
#    Settings → Secrets and variables → Actions → Update secret

# 3. Verify all workflows still pass

# 4. Revoke the old token immediately
#    Settings → Personal access tokens → Delete old token

# 5. Update this document with new rotation date
```

### 6. Emergency Revocation

If a token is suspected compromised:
1. **Revoke immediately** — do not wait for investigation
2. Follow Scenario 4 in the [Incident Response Runbook](INCIDENT-RESPONSE.md)
3. Generate a new token with the same (or narrower) scope
4. Update all references to the old token

## Workflow Token Permissions

Each GitHub Actions workflow should declare minimum permissions:

```yaml
# Good — explicit minimum permissions
permissions:
  contents: read
  actions: read

# Bad — implicit broad permissions
# (no permissions block = default read-write on everything)
```

## GitHub Actions `GITHUB_TOKEN` Security

- The `GITHUB_TOKEN` is automatically scoped per workflow run
- For fork PRs, the token has read-only permissions by default
- Never expose the `GITHUB_TOKEN` in logs or outputs
- Use `permissions` block to restrict even further
